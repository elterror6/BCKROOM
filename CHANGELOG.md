# CHANGE LOG

Todas las modificaciones importantes se documentarán es este archivo. En cada versión podrá haber cinco tipos de modificaciones:
- Added: Indica nuevas funcionalidades añadidas al proyecto.
- Changed: Indica un cambio de funcionalidades existentes.
- Fixed: Indica correciones de errores.
- Removed: Indica funcionalidades eliminadas.
- Deprecated: Indica funcionalidades marcadas como obsoletas.
Las nuevas versiones se pondrán encima de las antiguas separadas por una linea divisoria. Las versiones estarán representadas por un titulo de nivel 2 y con el formato [vM.m.p] - {fecha de salida dd/MM/YYYY}.
